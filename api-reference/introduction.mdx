The CasaOS REST API lets you programmatically interact with your CasaOS instance — checking system health, managing files, controlling storage, and more. All requests and responses use JSON.

<Note>
  The CasaOS API is only available on your local network. Use your device's IP address to connect (for example, `http://192.168.1.100/`). The API is not exposed to the internet by default.
</Note>

## API versions

CasaOS exposes two API versions, both served from the same base URL (`http://<your-casaos-ip>/`).

### v2 — current API

**Base path:** `/v2/casaos/`

The v2 API is the current, actively maintained API. It follows the OpenAPI 3.0 specification and includes request validation.

| Category | Endpoints |
|----------|-----------|
| Health   | `/v2/casaos/health/services`, `/v2/casaos/health/ports`, `/v2/casaos/health/logs` |
| Files    | `/v2/casaos/file/upload`, `/v2/casaos/file/test` |

### v1 — legacy API

**Base path:** `/v1/`

The v1 API covers a broader set of functionality. It remains available but is considered legacy.

| Category   | Base path           |
|------------|---------------------|
| System     | `/v1/sys/`          |
| Files      | `/v1/file/`         |
| Folders    | `/v1/folder/`       |
| Batch ops  | `/v1/batch/`        |
| Cloud      | `/v1/cloud/`        |
| Samba      | `/v1/samba/`        |
| Notify     | `/v1/notify/`       |
| Port       | `/v1/port/`         |
| Other      | `/v1/other/`        |
| Zerotier   | `/v1/zt/`           |

## Base URL

```
http://<your-casaos-ip>/
```

Replace `<your-casaos-ip>` with the IP address of the device running CasaOS on your local network.

## Request and response format

All API endpoints accept and return JSON. Set the `Content-Type` header to `application/json` for requests that include a body.

Responses follow a consistent envelope:

```json
{
  "message": "ok",
  "data": { ... }
}
```

The `message` field contains a human-readable status string. The `data` field contains the response payload and varies by endpoint.

## Example request

```bash
curl http://192.168.1.100/v2/casaos/health/services \
  -H "Authorization: <your-token>"
```

```json
{
  "message": "",
  "data": {
    "running": ["casaos-gateway.service", "casaos-app-management.service"],
    "not_running": []
  }
}
```
